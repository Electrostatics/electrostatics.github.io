David,

The patches CTERM and CYX in PATCH.xml are applyed correctly to Cysteine 17.

I checked the charge assignation and is OK (I compared the output PQR with
"grep CCYX AMBER.DAT").

There seems to be a simply issue in renaming CYS (17) to CYX. I also tried
with other forcefields or naming schemes and the results were the same
(e.g. using PARSE, CYS (17) was not renamed to CSS). The issue seems to be
related to the fact that it is a C-terminal CYS, because other CYS present
in the peptide are correctly renamed to CYX (even at the N-termini)

Thanks for your help

Matías

From: Matias Machado <matimachado@gm...> - 2015-09-14 19:07:44
Attachments: Message as HTML     
Dear APBS developers,

I may have found a bug in the pdb2pqr 2.0.0 release.

When processing the PDB id 1ETM either with the server or the standalone
version, pdb2pqr fails to correctly rename the C-terminal CYS 17 residue to
CYX. However this Cysteine is PATCHED to CYX as reported in the stdout.

Any guess why?

This is the command line:
pdb2pqr --ff=amber --ffout=amber --with-ph=7 --chain 1ETM.pdb 1ETM.pqr

Thanks for your answer!

Best regards,

Matías Machado

Re: [Pdb2pqr-users] PDB2PQR failure to rename C-terminal CYS to CYX
From: David Hall <dwash59_2000@ya...> - 2015-09-15 13:17:35
Attachments: Message as HTML     
Yes.

See http://www.poissonboltzmann.org/docs/pdb2pqr-programmers/ <http://www.poissonboltzmann.org/docs/pdb2pqr-programmers/>;

Click on "Atom and residue naming scheme”

Scroll down to "Amino Acid Residue Additions”

(Note to pdb2pqr developers, this section seems to be repeated twice?)


As for whether it is “reuse”, these names have long been used in the molecular modeling community to represent residue states. The PDB knows this, but decided to use these names any way decades later. Modelers have used CYX since at least 1982 ( http://pubs.acs.org/doi/abs/10.1021/ja00387a046 <http://pubs.acs.org/doi/abs/10.1021/ja00387a046>; ), but the PDB only started using it in 2003 and has only used it for a single structure.

-David
